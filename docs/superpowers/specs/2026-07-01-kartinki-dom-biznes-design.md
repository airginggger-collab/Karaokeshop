# Дизайн: картинки «караоке для дома / для бизнеса» — 2026-07-01

> Добавить сценарные изображения «дом» и «бизнес» в двух местах: секция-входы на главной + hero-фото на лендингах `/dlya-doma` и `/dlya-biznesa`.

## Цель

Дать визуальные сценарные входы «Караоке для дома» и «Караоке для бизнеса». При пересборке главной (2026-07-01) текстовые входы дом/бизнес убрали; возвращаем их как атмосферные карточки с фото и добавляем то же фото в hero лендингов (сейчас там только градиент).

## Источник изображений

**Сток Unsplash, тематические** (выбор владельца). Ключевое: фото должны быть *релевантны сценарию* (предупреждение CLAUDE.md касается нетематического стока):
- **Дом:** уютная гостиная / вечер с песнями / семья, тёплый свет.
- **Бизнес:** ресторан / бар / клуб вечером, атмосфера заведения.

**2 файла**, переиспользуются в обоих местах:
- `apps/web/public/scenariy/dom.webp`
- `apps/web/public/scenariy/biznes.webp`

Скачиваются с Unsplash и конвертируются в webp через `sips` (вес). **Скачивание — только после явного подтверждения:** на этапе реализации показать конкретные фото-кандидаты (URL + размер), дождаться «ок», затем закачать. Формат/размер ориентир: ландшафт ~1200×800, webp.

## Что делаем

### 1. Секция на главной — «Дом и бизнес» (две карточки с фото)

- Две карточки в ряд (desktop), стопкой (mobile). Каждая: фото (верх/фон), заголовок, подзаголовок, CTA.
  - «Караоке для дома» → `/dlya-doma`, подзаголовок про гостиную/баню/гостевой дом, CTA «Выбрать домой».
  - «Караоке для бизнеса» → `/dlya-biznesa`, подзаголовок про кафе/бар/клуб под ключ, CTA «Оснастить заведение».
- **Размещение:** в `apps/web/src/app/page.tsx` — после секции «Как мы работаем», перед «Дилерской полосой». Квиз в hero остаётся главным CTA; эта секция — атмосферный вход «для кого мы».
- Рендер фото: `<img>` с `loading="lazy"`, `object-cover`, фиксированной высотой (напр. `h-48 sm:h-56`), скруглением карточки. Ссылка-обёртка `<Link>`.

### 2. Hero-фото на лендингах

- `apps/web/src/app/dlya-doma/page.tsx` и `apps/web/src/app/dlya-biznesa/page.tsx`: в существующий hero-`<section>` (сейчас градиент `from-surface to-muted`) добавить изображение — сбоку на desktop (двухколоночный hero), сверху на mobile. Использовать те же `dom.webp` / `biznes.webp`.
