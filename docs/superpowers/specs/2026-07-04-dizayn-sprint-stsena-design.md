# Дизайн-спринт «Сцена» — де-AI-фикация всего сайта

> Спек утверждён владельцем 2026-07-04 (брейнсторм: аудитория B2C-дом первична; якорей бренда нет — можно менять всё; объём — весь сайт; из трёх директур выбрана A «Сцена»; секции 1–3 утверждены по отдельности).

## Цель

Убрать узнаваемую «AI-эстетику» (шаблонный hero, ряды pill-чипов, счётчики, одинаковые rounded-2xl карточки, сток-фото) и дать сайту собственную арт-директуру, пригодную для показа живым людям и инвесторам. Это **шестая и последняя смена акцента** — предыдущие пять (тил → янтарь → терракот → синий → графит/неон) были перекрасками без директуры; эта — система, из которой следуют все дальнейшие визуальные решения.

## Арт-директура «Сцена»

Караоке = ночь концерта. Тёмная база — не «dark mode ради dark mode», а сцена: глубокий фиолетово-чёрный фон, свет падает на главное (заголовок, CTA, товар). Фиолетовая марка лого `#7c5cff` перестаёт быть «приклеенной» — весь сайт говорит на её языке.

**Фирменный приём — «подсветка строки».** Как в караоке подсвечивается текущая строка текста: ключевая строка/слово заголовка получает плоскую заливку акцентом (`--color-cta`: `#7250f8` dark / `#6742e8` light, текст белый). При появлении в вьюпорте заливка заполняется слева направо (~0.6s, ease-out); `prefers-reduced-motion` — сразу финальное состояние; SSR отдаёт финальное состояние (работает без JS). Приём переиспользуется системно: активный пункт навигации, прогресс квиза, ховер ссылок «Подробнее». Один язык вместо россыпи механик.

- Неон-жёлтый `#facc15` и графит-акцент удаляются. Марка лого не меняется.
- `--color-hot` — строго дозирован: скидки и «горячие» бейджи, максимум одно пятно на экран, **не для кнопок**.
- Каждая пара текст/фон — минимум AA (4.5:1); проверяется на шаге токенов. Фиолетовый на тёмном — только акценты; длинный текст — нейтральный `--color-fg`/`--color-muted-fg`.
- `.mood-*`-классы и инлайн-использования `--warm-*`/`--night-*` (главная, dlya-biznesa, gotovye-resheniya, HeroWave) переводятся на новые токены; `--warm-*`/`--night-*` после миграции удаляются из `theme.css`.
- Радиусы (`packages/tokens/tokens.json` → Style Dictionary): карточки 12px, кнопки/поля 10px. `rounded-3xl` запрещён.

## Типографика

Пара Manrope (текст) + Unbounded (дисплей) остаётся — меняется смелость использования:
- h1: Unbounded 700, крупнее текущего (клэмп ~2.5–4rem), line-height 1.05–1.15, одна строка с подсветкой.
- Заголовки секций: номер `01/02/03` Unbounded 700 приглушённым `#453a6e` + заголовок; иконки-кружки у заголовков запрещены.
- Цифры доверия — спокойной типографикой в строке-тикере (см. ниже), без CountUp.

## Анти-AI правила (обязательные; после спринта добавить в CLAUDE.md)

1. **Ни одного ряда pill-чипов.** Факты доверия — одной строкой-тикером вдоль тонкой линии: `официальный дилер AST · Studio Evolution · Алматы · монтаж под ключ` (letter-spacing .12em, muted).
2. **CountUp-счётчики уходят с витрин.** Компонент и тесты остаются в репо; цифры в UI — только рядом с доказательством (кейс/документ), без анимации роста.
3. **Сетка «иконка + заголовок + текст» — максимум одна на страницу**, оформление через нумерацию, не через lucide-кружки.
4. **Тени-«клей» убираются.** Разделение поверхностей: цвет (`--color-bg` на `--color-page`) + hairline `1px solid var(--color-border)`. Тени в `globals.css` (clay-shadow) удаляются.
5. **Сток-фото (Unsplash) удаляются все и сразу** (`public/scenariy/*`): до реальных фото — типографический плейсхолдер (плашка `#1f1832`/light `#e5e0f2` с крупным именем модели/сцены в Unbounded, приглушённым тоном). Плейсхолдер честный — не притворяется фотографией.
6. **Фото товаров** — на нейтральной сцене-подложке (`#1f1832` dark / `#f0eef8` light с паддингом), белые плашки в тёмной теме запрещены. Реальные фото при поступлении — при необходимости фиолетовый дуотон для унификации качества.
7. **Асимметрия вместо симметрии:** hero и крупные секции — сдвиг заголовочного блока, перекрытие фото-блока и текстовой колонки на `lg+`. Секции чередуют полосы `--color-page`/`--color-bg` («сцены»).
8. **Один WhatsApp-CTA на экран** (вьюпорт). Иерархия кнопок: зелёный WhatsApp (primary-действие связи) → фиолетовый CTA (действие на сайте) → ghost (вторичное).
9. **Motion:** один оркестрованный вход hero (stagger появления строк) + заполнение подсветки; ховеры карточек — смена цвета заголовка на `--color-primary`, без подъёмов/scale. `prefers-reduced-motion` соблюдается везде.

## Порядок реализации

Каждый шаг: `npm run build -w web` + `npm test -w web` (24) + превью обеих тем (`kk-theme`) + коммит с HANDOFF + пуш + `gh run` (CI зелёный):

1. **Токены**: `theme.css` (палитра выше), `tokens.json` (радиусы), `tailwind.config.ts` (маппинг `hot`, `border`), чистка `globals.css` (тени). Сайт «перекрашивается» целиком.
2. **Шелл + UI-кит**: Header/Footer/MobileNav/Button/Badge, компонент подсветки строки (`HighlightLine`), строка-тикер.
3. **Главная**: пересборка по спеку.
4. **Каталог + товар.**
5. **Лендинги** (миграция mood-инлайнов, плейсхолдеры вместо стока).
6. **Блог + служебные + удаление `--warm-*`/`--night-*` и HeroWave-орбов**, финальный проход по хардкодам (`grep 'text-\[#' 'bg-\[#'` — допустимы только `#25D366` и `#7c5cff` марки).

## Риски

- **Контраст светлой темы**: фиолет `#6742e8`/`#4c30b8` проверить на `#f4f2fa` и `#ffffff` (AA). Проверка на шаге 1, до раскатки.
- **Whack-a-mole тем**: не вводить ни одного `dark:text-[#…]` — только токен-утилиты (правило репо).
- **Удаление стока** заденет hero главной/dlya-doma/dlya-biznesa — плейсхолдеры входят в тот же шаг, страницы не остаются «дырявыми».
- **LCP**: убирание тяжёлых jpg (290K) из hero скорее улучшит Lighthouse; следить за бюджетом (warn ≥0.9).
