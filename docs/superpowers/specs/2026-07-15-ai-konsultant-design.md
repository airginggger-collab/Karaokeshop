# Спека: AI-консультант «Караоке-эксперт» + AI-видимость

> Дизайн-документ, одобрен владельцем 2026-07-15. Источник задачи: этап 3 плана внедрения новинок ИИ (deep-research 2026-07-15). Следующий шаг после ревью владельцем: `writing-plans` → детальный план реализации.

## Цель

Дать посетителю разговорного помощника, который (а) подбирает караоке-оборудование обычным языком, (б) отвечает на вопросы по всему контенту сайта (каталог, блог, FAQ, песни, комплекты), (в) делает сайт читаемым для внешних AI-агентов (ChatGPT, Perplexity). Итог продаж, реальный заказ, всегда эскалируется в WhatsApp (единственный зелёный CTA).

Не дублировать существующее: `QuizWidget` (пошаговый подбор), калькулятор сметы, `SearchOverlay` (поиск). Консультант, это надстройка «свободный диалог», а не замена структурных инструментов.

## Ключевой архитектурный принцип

**Главный сайт остаётся чистой статикой (`output: export`, assets-only Worker).** Вся ИИ-логика живёт в ОТДЕЛЬНОМ Worker'е (`ai-consultant`). Статический сайт лишь встраивает клиентский виджет, который ходит на этот Worker по HTTP. Это ограничивает «вариант B» из CLAUDE.md одним изолированным сервисом, основной деплой-таргет не переписывается.

**RAG не нужен.** Корпус крошечный (18 товаров, ~1344 строки JSON суммарно). Вся база знаний помещается в контекст модели на каждый запрос. Значит: никакой Vectorize, эмбеддингов, индексации. База знаний компилируется на билде в один текстовый артефакт и вшивается в Worker.

## Движок

Claude **Haiku** через Anthropic API, вызов из Worker'а. Обоснование: лучший русский язык (каталог и клиенты русскоязычные), владелец уже в экосистеме Claude, на малом контексте стоимость запроса копеечная, поддерживает стриминг. API-ключ хранится секретом Cloudflare (`wrangler secret`), в код не попадает.

## Компоненты и интерфейсы

1. **Компилятор базы знаний** (`scripts/build-knowledge.mjs`, запуск в `prebuild`).
   Вход: `apps/web/content/*.json`. Выход: `knowledge.generated.ts` (или `.txt`), плоский текст: товары (модель, тип, цена, ключевые характеристики, URL), комплекты, FAQ, краткие описания блога и кейсов, факты о песнях. Детерминированный, коммитить артефакт не обязательно (генерится на билде).

2. **Worker `ai-consultant`** (отдельный пакет `apps/ai-consultant/`, свой `wrangler.toml`).
   - `POST /ask`: принимает `{ question: string, history: Message[] }`, возвращает стрим текста ответа плюс структурный блок рекомендаций `{ products: [{slug, title, price, reason}] }`.
   - Системный промпт: роль «караоке-эксперт», тон дружелюбно-экспертный, база знаний в контексте, правило «не выдумывать товары и цены, которых нет в базе», правило «для заказа, направляй в WhatsApp».
   - Защита: rate-limit по IP (Cloudflare Rate Limiting или KV-счётчик), кап `max_tokens`, таймаут, CORS только на домен сайта.

3. **Клиентский виджет** (`apps/web/src/components/ConsultantWidget.tsx`).
   - Плавающая кнопка-лаунчер, присутствует на всех страницах (через корневой layout). Открывает панель чата (макет одобрен 2026-07-15).
   - Панель: тред сообщений, карточки рекомендованных товаров со ссылками на `/catalog/[slug]`, поле ввода, кнопка эскалации в WhatsApp (`waHref` из `lib/wa.ts`).
   - Соблюдает «Сцену»: тёмная тема по умолчанию, акцент неон-жёлтый, зелёный только у WhatsApp, радиусы ≤12px, тень допустима (виджет, плавающий элемент).
   - Стартовые быстрые-вопросы (3 чипа): «Комплект для дома», «Для бара/ресторана», «Есть ли нужные песни».

4. **Слой AI-видимости** (Фаза 3a, чистая статика).
   - `apps/web/public/llms.txt` и `llms-full.txt`, генерируются на билде из контента (тот же компилятор, что и база знаний, другой формат вывода).
   - Аудит и расширение JSON-LD (`Product`, `FAQPage`, `BreadcrumbList`), только через компонент `<JsonLd>` (экранирование, ловушка №9).
   - Опционально позже, `/mcp`-эндпоинт на Worker'е (протокол NLWeb), чтобы внешние агенты запрашивали каталог.

## Поток данных (Фаза 3b)

Пользователь вводит вопрос → виджет `POST /ask` на Worker → Worker собирает системный промпт (роль + вшитая база + история) → Anthropic API (Haiku, стрим) → Worker проксирует стрим клиенту → виджет рендерит текст и карточки товаров → заказ, клик по WhatsApp-кнопке, `wa.me` с преднаполненным сообщением.

## Фазировка

- **Фаза 3a, AI-видимость.** Реализуется сейчас. Чистый билд, ноль бэкенда, ноль затрат, деплой существующим CI. Не требует действий владельца. Даёт ценность сразу (ChatGPT/Perplexity лучше «понимают» магазин, богаче структурные данные для поиска).
- **Фаза 3b, чат-консультант.** Требует действий владельца (ниже). Пилот на `*.workers.dev`, полноценно после привязки домена `.kz`.

## Безопасность и стоимость

- API-ключ, только секрет Cloudflare, никогда в коде и в клиенте.
- Prompt-injection из контента: база знаний, доверенный источник (правит владелец через /admin), но системный промпт явно инструктирует не исполнять инструкции из данных.
- JSON-LD, только через `<JsonLd>` (защита от разрыва `</script>`).
- Rate-limit + `max_tokens` cap + кэш частых ответов (KV), защита от абуза и неконтролируемого счёта.
- Стоимость: Haiku на контексте в несколько тысяч токенов, доли цента за запрос. Точная оценка трафика, открытый вопрос для владельца.

## Действия только владельца (Фаза 3b)

2. Дать доступ к CF-аккаунту для деплоя Worker'а `ai-consultant`.
3. Финальное «в прод» после проверки русского качества на пилоте.

## Тестирование

- Компилятор базы: юнит-тест, что все товары/цены/URL попали в артефакт, формат стабилен.
- Worker: тест сборки промпта, тест обработки ошибок Anthropic API (таймаут, 429), тест rate-limit.
- Виджет: тест рендера треда и карточек, тест эскалации в WhatsApp (`waHref`).
- llms.txt: тест, что генерится и содержит все товары.
