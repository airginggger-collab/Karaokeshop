## Партия A — фундамент (1 агент, зависимостей нет)

Создать и покрыть простыми тестами (где есть логика):

1. **`src/lib/wa.ts`**: `export function waHref(text?: string): string` — собирает `https://wa.me/<номер>` + опциональный `?text=` через `encodeURIComponent`. Мигрировать сюда 21 сборку. Тест: 2–3 кейса (без текста, с текстом, спецсимволы).
2. **`src/components/WaButton.tsx`** (SSR): зелёный анкор WhatsApp. Пропсы: `text?` (сообщение), `children` (лейбл), `size?: "md"|"lg"` (паддинги встречаются `px-5 py-3` и `py-3.5`), `className?`, `full?` (checkout/calculator — во всю ширину). Внутри `waHref()`, `target="_blank" rel="noopener noreferrer"`, hover `#1ebe5d`. Иконку передавать children'ом. НЕ трогать WhatsAppFAB-стили (плавающая кнопка остаётся отдельной, но href берёт из `waHref`).
3. **`src/components/SectionHeader.tsx`** (SSR): `title` (ReactNode), `sub?`, `action?: { href, label }` — рендерит h2 (`font-display text-xl font-semibold`; на главной размер крупнее — проп `size`) и **двойную** ссылку (desktop `hidden sm:flex` в шапке + mobile `flex sm:hidden` после секции — вторую часть рендерит сосед; см. ниже вариант: `SectionHeader` отдаёт только шапку, а mobile-ссылку — компонент `MobileActionLink`; если получается громоздко — оставить двойную разметку внутри одного компонента-обёртки `<Section>` с `children`).
4. **`src/components/CtaSection.tsx`** (SSR): финальный CTA-блок. Пропсы: `title`, `text`, `waText`, `waLabel?`, `secondary?: { href, label }`, `note?`, `ticker?`. Сверить все 8 вхождений и покрыть их вариации пропсами; если у страницы уникальная структура (главная с квизом) — не натягивать, оставить как есть.
5. **`src/components/FaqBlock.tsx`** (SSR): `faq: {q,a}[]`, `title?` — рендерит `<JsonLd data={faqJsonLd(faq)}/>` + секцию с `FaqAccordion`. JSON-LD и видимый FAQ никогда не рассинхронизируются.
6. **`src/components/OptionButton.tsx`** (client-safe, без хуков): кнопка-опция с `selected`. Использовать в CalculatorClient (шаги 1, 3) и QuizWidget.
7. **`Breadcrumb`**: добавить опциональный проп `withLd?: boolean` — компонент сам рендерит `breadcrumbJsonLd` из своих items (плюс «Главная»). Страницы, где items крошки и JSON-LD совпадают, переходят на `withLd` и выкидывают ручной `<JsonLd data={breadcrumbJsonLd(...)}/>`. Проверить, что Breadcrumb — SSR-компонент (без "use client"); если клиентский — вместо пропа сделать обёртку `BreadcrumbWithLd` (SSR).

Критерий: `npm run build -w web` + `npm test -w web` зелёные. Ничего из существующих страниц ещё не мигрировано (компоненты добавлены, но не используются) — допустимо закоммитить фундамент отдельно.

## Партия C — UX-фиксы пути клиента (после B; итоги прогона 375px, локальный билд = HEAD)

> Прогон: главная → квиз (до результата) → поиск → каталог → товар AST-250 → корзина → checkout → WhatsApp; калькулятор до сметы; мобильное меню; обе темы. Что работает хорошо: квиз в hero доводит до цены и заявки, sticky-бар товара, checkout с корректным итогом, 44px-кнопки в квизе/чекауте, мобильное меню читабельно в обеих темах.

**C1 — КРИТИЧНО. На мобильном товаре нет «В корзину».** CTA-блок в `product/[slug]/page.tsx` — `hidden lg:flex` (строка ~121), в `ProductStickyBar` только WhatsApp. Корзина и checkout недостижимы с телефона (80% трафика): иконка корзины в шапке ведёт в пустую корзину. Фикс: в `ProductStickyBar` добавить компактную кнопку «В корзину» (вторичная, рядом с WhatsApp-CTA; после добавления — состояние «В корзине ✓» со ссылкой на /checkout). Иерархия CTA сохраняется (WhatsApp главный).

**C2 — Поиск обрывается на Enter.** `SearchOverlay` сабмитит `/catalog?q=…`, но `CatalogClient` не читает `useSearchParams` — клиент видит все 18 товаров и пустой инпут, думает «поиск сломан». Фикс: `CatalogClient` инициализирует `q` из `?q=` (клиентский компонент; обернуть в `<Suspense>` по требованию Next для useSearchParams; статик-экспорту не мешает).

**C3 — Поиск не находит товары по типу.** Запрос «микрофон» в оверлее и в каталоге даёт 0 товаров (ищут только `model`/`brand`/`scenarioLabel`), хотя микрофоны есть. Фикс: включить в матчинг `typeLabels[p.type]` и `note` (оба места: `SearchOverlay`, `CatalogClient`).

**C4 — Checkout теряет корзину при неудачном открытии WhatsApp.** `CheckoutClient.handleSubmit`: `clear()` вызывается ДО `window.open(wa.me…)`. Фикс: сначала `window.open`, потом `clear()+setDone` (открытие остаётся в обработчике клика — попап-блокер не мешает).

**C5 — Результат квиза обезличен.** «Караоке-система (компакт)» без модели и ссылки. Мини-фикс: под результатом добавить ghost-ссылку «Посмотреть системы в каталоге» → `/catalog` (глубже не лезть: маппинг сметы на товары — отдельная задача).

**C6 — Мёртвая кнопка «Нужна консультация»** (`product/[slug]`, десктоп-CTA): `<Button variant="ghost">` без действия. Фикс: `WaButton`-ghost со своим текстом («Здравствуйте! Нужна консультация по …») или удалить. Предпочтительно первое.

**C7 — Телефон некликабелен в мобильном меню.** `MobileNav.tsx:104` — просто `<p>`. Фикс: номер обернуть в `<a href="tel:…">` (паттерн уже есть в Footer).

**C8 — Фото-зона товара на мобильном — огромный квадрат** (`aspect-square` на 375px + временные фото с белым фоном = экран пустоты до цены). Фикс: `aspect-[4/3]` на мобильном (`lg` не трогать). Фото временные (ловушка 4), «белую плашку» не чинить сейчас.

**C9 — Остатки «—» в видимой прозе:** подсказки строк сметы калькулятора (`lib/components.ts`, hint'ы: «…эхо-процессор — ключевой модуль», «АЧХ до 40 Гц — голос звучит объёмнее») — правило №10, зачистка их пропустила (были только по страницам/контент-JSON). Заменить с вариацией.

**C10 — Тап-таргеты фильтров каталога 32px** (чипы типов). Фикс: `min-h-10`+паддинги (≥40px), не меняя вид радикально.

## Порядок и проверки

1. Партия A → build+test → коммит `refactor(ui): фундамент дедупликации (wa.ts, WaButton, CtaSection, FaqBlock, OptionButton, SectionHeader, Breadcrumb withLd)`.
2. Партия B (3 агента параллельно) → build+test → сверка живых страниц (`npx serve apps/web/out`, обе темы) → коммит `refactor(pages): миграция на общие компоненты, минус ~N строк`.
3. Партия C → build+test → коммит `fix(ux): …`.
4. Каждый коммит: обновить `docs/HANDOFF.md`; пуш сразу; `gh run list` до success (ловушка 0).
5. Метрика успеха: суммарно −25…−35% строк в `apps/web/src` без визуальных регрессий; все 29+2 теста зелёные; Lighthouse SEO ≥ 0.95 не падает.

## Ограничения (не делать)

- Не трогать `content/*.json` (кроме случаев из партии C, если потребуется текст CTA).
- Не менять `LandingPage`-механику `/karaoke/*`, `/komplekty/*` без необходимости.
- Не «улучшать» дизайн-систему: токены, радиусы, палитра — вне скоупа.
- Не возвращать Kaspi/рассрочку, счётчики/сток-фото не убирать (решения владельца).
- SHOW_UNVERIFIED_* флаги и скрытый SocialProof не удалять.
