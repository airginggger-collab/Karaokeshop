# context/ — контекст для передачи разработчикам (handoff)

То, что отдаётся бэк/фронт-командам. Накапливается по мере Фазы 1.

**Что здесь будет:**
- `data-model.md` — сущности, связи, инварианты (product, bundle, case, song, page).
- `api-contracts/` — OpenAPI-спеки для бэка (`/api/v1/...`).
- `content-model.md` — схема CMS (зеркало `packages/cms-schema`), SEO- и i18n-поля.
- `events.md` — событийная схема аналитики (`view_item`, `add_to_cart`, `lead_submit`, `configurator_use`) + лог внутреннего поиска.
- `env.example` — переменные окружения.

**Принцип:** документ должен позволить разработчику с нулевым контекстом понять систему. Файлы, которые меняются вместе, — рядом. Источник истины для модели — `packages/cms-schema`, для токенов — `packages/tokens`; здесь — их человекочитаемое описание и контракты.
