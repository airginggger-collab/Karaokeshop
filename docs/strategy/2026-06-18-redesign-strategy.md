# Karaokeshop.kz — стратегия полного редизайна

> Документ-основа проекта. Анализ рынка → позиционирование → архитектура → стек → дорожная карта.
> Дата: 2026-06-18 · Контекст: фриланс/портфолио-ребилд · Источник данных: публичные (внутренних метрик нет).

---

## 0. TL;DR

`karaokeshop.kz` — магазин + интегратор караоке-оборудования в Алматы (бренды **AST** и **Studio Evolution / Evobox**, чеки ~200 тыс — 2,5 млн ₸). Задача — **полный ребилд** под четыре цели одновременно: онлайн-продажи (B2C), B2B-лиды (оснащение заведений), имидж/бренд и **SEO как главный приоритет** (находиться в поиске лучше конкурентов).

**Ключевые решения:**
- **Архитектура — модель B:** единая витрина с сегментацией по намерению (для дома / кафе / бара / клуба) + выделенная B2B-воронка «под ключ». Не два отдельных сайта — это консолидирует SEO-авторитет.
- **Стек:** Next.js (SSR/SSG) + Headless CMS. Под SEO, скорость, портфолио и самостоятельное ведение контента владельцем.
- **SEO — позвоночник:** семантическое ядро проектируется ДО структуры; страница-под-намерение; двойная оптимизация Google + Яндекс; микроразметка; локальное SEO Алматы.
- **i18n-ready:** RU сейчас, KZ-контент позже (но hreflang/структура закладываются сразу — незанятая ниша, все конкуренты только на русском).
- **Платформа на вырост:** версионность по слоям, админка для самостоятельного контента, handoff-инфраструктура для передачи разработчикам.

## 2. Анализ рынка Казахстана

Прямой цифры «рынка караоке-оборудования» в KZ не существует — используем макропрокси (с оговорками).

- **Индустрия развлечений: 523,3 млрд ₸ в 2025, +36,5% YoY** (после −25,9% в 2024) — рекорд. *Caveat: широкий досуг (парки, аниматоры, автоматы), не караоке специфически.* [kapital.kz]
- **HoReCa (адресуемый B2B): 16 674 заведения общепита в 2024, +5,4% YoY.** Лидеры: Алматы 2 427, Астана 2 252, Карагандинская обл. 1 333. *Caveat: ~5 618 из них — столовые (не покупатели караоке); по ресторанам Астана впереди Алматы (524 vs 446).* [kaztag.kz]
- **~156–158 караоке-заведений в Алматы** (каталог Restoran.kz) — плотный локальный рынок замены/апгрейда.

**Вывод:** спрос восстанавливается уверенно; B2B-ядро сконцентрировано в Алматы+Астане; компания в правильном городе.

**Отсеяно при проверке (недостоверно):** тезис «рынок караоке вырос на 60% за 5 лет» (PR-заявление вендора, рефутирован 0-3).

## 3. Конкуренты и продукт

**Два бренда правят рынком** — и их линейки уже разделены по сценарию (готовая ось для IA):
- **AST (Art System):** HOME / Zoom / Mini — «для квартиры, дома, небольшого кафе»; AST-250 / 350 — «для караоке-клубов, баров, ресторанов».
- **Studio Evolution:** Evobox (дом) → Evobox Plus / Pro2 (заведения).

**Ценовая лестница (снимок; скидки 7–44% — норма, цены курсо-/сезонно-зависимы):**

| Сегмент | Модель | Цена |
|---|---|---|
| B2C вход | AST HOME | от ~415 000 ₸ |
| B2C | AST Mini / Evobox | 720 000 / 749 000 ₸ |
| Средний | AST-50 | ~960 тыс — 1,32 млн ₸ |
| Премиум B2B | Evobox Plus | 1 390 000 ₸ |
| Премиум B2B | AST-250 | 2 664 000 → **1 500 000 ₸ (−44%)** |
| Премиум B2B | Evolution Pro2 | 1 790 000 ₸ |
| Топ B2B | AST-350 | по запросу |

**Конкуренты-ритейлеры:**
- `1music.kz` («Первый музыкальный») — «для дома, кафе и ресторана», бесплатная доставка по KZ + СНГ. Единый каталог (подтверждает модель B).
- `art-system.kz` — фирменный магазин AST + сервис-центр.
- `karaoke-shop.kz` — ArtSystem + Studio Evolution.

**Ключевой B2B-дифференциатор — сервис «под ключ», а не цена:** проектирование звука под помещение → монтаж → настройка → обучение персонала → обновление репертуара (пакет по договору). Кто яснее покажет сервис — забирает дорогой чек.

## 4. Аудитория и путь покупателя

> Выведено из позиционирования вендоров; не подкреплено первичными опросами (см. открытые вопросы).

**B2B (владелец заведения):** длинный цикл, дорогой чек. Факторы решения: проект под помещение, монтаж, гарантия, обучение, обновление песен, кейсы похожих заведений, бренд. Главный страх — «что мне вообще нужно». → закрывается **подбором по площади + расчётом + заявкой с консультацией**.

**B2C (домой):** короткий цикл, объём. Факторы: цена и **рассрочка**, число песен, простота, доставка, отзывы. → закрывается **быстрым e-commerce + Kaspi**.

## 5. E-commerce и оплата (специфика KZ)

- **Kaspi.kz доминирует:** маркетплейс GMV +44% YoY (2024), **72 транзакции/мес на пользователя**, ~6 000 собственных постаматов; рассрочка/Kaspi Pay — привычный паттерн. *Нюанс: доля Kaspi в e-comm снижается с ~77% до ~70% (растут Ozon/WB), но он остаётся №1.* [SEC-филинг Kaspi]
- **Вывод:** интеграция Kaspi (рассрочка/Kaspi Pay в checkout) + **параллельная витрина в Kaspi Магазин** как второй канал обязательны. Дорогой B2B-чек всё равно закрывается заявкой + консультацией.
- **Отсеяно:** «рассрочка/BNPL — главный механизм покупки» (рефутирован 1-2). Правдоподобно для дорогих систем, но данными не подтвердилось — не строим стратегию только на рассрочке.

## 6. SEO-стратегия (главный приоритет)

**Конкуренты уязвимы по трём фронтам — бьём в каждый:**

1. **Техническая база (Next.js > их платформы):** SSR + зелёные Core Web Vitals + чистые ЧПУ + sitemap + mobile-first. Старые движки конкурентов медленнее — прямой фактор Google и Яндекс.
2. **Двойная оптимизация Google + Яндекс:** Google ~69%, но **Яндекс №2 и растёт — >24%** (StatCounter 24,8% 2024 → 28,7% к 2026), конкуренты его игнорируют. Яндекс ранжирует по поведенческим факторам → наш UX-фокус прямо качает позиции. Яндекс.Вебмастер + Метрика + работа над ИКС.
3. **Страница-под-намерение (главный рычаг):** отдельная оптимизированная посадочная под каждый кластер; фасеты каталога — индексируемые лендинги (`/karaoke/dlya-bara`, `/karaoke/ast`) с canonical-дисциплиной.
4. **E-E-A-T контент** (у конкурентов пусто): гайды, сравнения AST/Evolution, FAQ → топикальный авторитет.
5. **Микроразметка Schema.org:** Product, Offer, AggregateRating/Review, BreadcrumbList, FAQPage, LocalBusiness → богатые сниппеты → выше CTR.
6. **Локальное SEO Алматы:** Яндекс.Бизнес + Google Business Profile + 2ГИС, единый NAP, гео-лендинги.

### Черновое семантическое ядро → карта посадочных
> Частоты — из Яндекс.Wordstat / Google Keyword Planner на Фазе 0. Здесь — структура по намерению.

| Кластер | Примеры | Целевая страница (URL) | Приоритет |
|---|---|---|---|
| B2C сценарий | караоке для дома, для квартиры, домашняя система купить | `/karaoke/dlya-doma` | 🔴 выс |
| B2B сценарий | караоке для бара, ресторана, кафе, для клуба | `/karaoke/dlya-bara` … | 🔴 выс |
| Под ключ | оснащение караоке клуба под ключ, под ключ Алматы | `/pod-klyuch` | 🔴 выс |
| Гео | караоке система купить Алматы, оборудование Алматы | гео-модиф. + Я.Бизнес/2ГИС | 🔴 выс |
| Бренд/модель | AST-250 цена, Evobox купить, AST Mini, Pro2 | карточки + `/brand/*` | 🟠 сред |
| Сравнение | AST или Evolution что лучше, рейтинг систем | статья-сравнение | 🟠 сред |
| По типу | микрофоны, радиомикрофон, акустика, микшер, свет | категории каталога | 🟠 сред |
| По площади | для бара до 80 м², комплект для ресторана | лендинги комплектов | 🟢 хвост |
| Информационные | как выбрать караоке для бара, сколько стоит оснастить | блог/гайды | 🟢 верх воронки |
| Сервис | настройка, обновление песен, ремонт, гарантия | `/servis`, `/pesni` | 🟢 хвост |

**Незаняты конкурентами:** сценарные, гео+под ключ, по площади, весь E-E-A-T-слой. Берём долю там. KZ-зеркало через hreflang — вторая незанятая плоскость.

## 7. Позиционирование

**«Караоке под ключ в Казахстане — от системы домой до оснащения клуба».** Экспертность (с 2012) + сервис на витрине, а не «магазин коробок». Бренды AST/Evolution как знаки качества; каталог песен, гарантия, монтаж, кейсы — сквозные элементы доверия.

## 8. Архитектура сайта (IA) — модель B

**Сравнение моделей:**
- **A — жёсткая развилка на главной:** ясно, но главная ≠ главный вход (трафик из поиска падает на товар), раздваивает SEO и поддержку. ❌
- **B — единая витрина + B2B-трек (выбрано):** один каталог с сегментацией по намерению + постоянный трек «под ключ». Консолидирует SEO, совпадает с реальным путём покупателя, так работает лидер 1music. ✅
- **C — B2B-first интегратор:** премиально, но слабо для B2C-объёма. ❌

**Навигация:** Каталог (мега-меню: по сценарию / по типу / по бренду) · Готовые комплекты (по площади) · **Под ключ** · Песни · Сервис · О нас · Контакты · Поиск · Корзина · бейдж Kaspi.

**Типы страниц:** Главная (двухпутевой герой) · Каталог/листинг (фасеты) · Карточка системы · Готовый комплект · B2B «под ключ» (калькулятор по площади + кейсы + заявка) · Каталог песен · Сервис · О нас · Контакты · Корзина/checkout.

## 9. Прототипы экранов

Собраны 4 структурных прототипа (flat-стиль; визуальный бренд — в Figma на этапе дизайна):
1. **Главная** — двухпутевой герой (домой / заведение), знаки доверия, рассрочка Kaspi на карточках, сценарий-теги.
2. **B2B «Под ключ»** — 5 этапов сервиса + рабочий калькулятор «площадь × тип → комплект + цена → заявка с собранным контекстом».
3. **Каталог** — фасетные фильтры (сценарий, бренд, площадь, цена) + сетка карточек со сценарий-тегами.
4. **Карточка системы** — галерея, цена+рассрочка, что в комплекте, двойной CTA (купить / консультация), гарантия/монтаж/песни, отзывы.

## 10. Технологический стек (ADR)

**Решение: Next.js (App Router, SSR/SSG) + Headless CMS.**

| Критерий | Кастом Next.js | **Next.js + Headless CMS** ✅ | Готовая (Tilda/Bitrix) |
|---|---|---|---|
| SEO/скорость | ★★★ | ★★★ | ★ |
| Владелец ведёт контент | ✗ | ★★★ | ★★★ |
| Гибкость UX | ★★★ | ★★★ | ★ |
| Скорость старта | ★ | ★★ | ★★★ |
| Сила как кейс | ★★★ | ★★★ | ★ |

**Обоснование:** SSR + structured data + быстрые CWV → ранжирование и портфолио; CMS снимает возражение «владелец не обновит кастом». Коммерция — лёгкий слой (Medusa/Snipcart или commerce в CMS), полноценный Shopify избыточен; ключевой нюанс — **Kaspi**. Хостинг — Vercel; обязательны Яндекс.Вебмастер + GSC + Метрика.

**ADR-открыто (решить в Фазе 0/1): CMS — Sanity vs Strapi.**
- Критерии: резидентность данных в KZ, бюджет DevOps, скорость старта, качество i18n и истории версий.
- Предварительно: **Sanity** (хостинг, мощная история версий, real-time, i18n) при отсутствии требования резидентности; **Strapi** (self-hosted, свои данные в SQL) при требовании контроля/резидентности.

## 12. Админка / самостоятельное обновление контента

**Headless CMS = админка с первого дня.** Владелец/маркетолог без разработчика правит: товары, цены, комплекты, B2B-кейсы, каталог песен, блог, страницы, меню, 301-редиректы, **SEO-поля на каждой странице** (title/description/og/canonical), поля **RU/KZ** в одной записи. Роли: владелец / редактор / маркетолог.

**Дорожная карта админки:** Фаза 2 — CMS Studio (контент). Фаза 5+ — опциональный кастом-дэшборд (заказы, B2B-заявки, аналитика). Модель данных проектируется так, чтобы дэшборд можно было построить позже без переделки ядра.

## 13. Инфраструктура и handoff-готовность

**Монорепо (Turborepo):**
```
apps/web            — Next.js фронт
packages/ui         — компоненты + Storybook
packages/tokens     — дизайн-токены (Figma → код)
packages/cms-schema — модель контента как код
packages/config     — eslint/ts/tailwind общие
docs/               — контекст-пак
```

**Контекст-пак `/docs/` (отдаётся разработчикам):** `data-model.md`, `api-contracts/` (OpenAPI), `content-model.md`, `design-system.md` + Storybook + Figma Code Connect, `adr/`, `seo-spec.md`, `env.example`, CI-конфиги.

**Сбор данных для контекста:** аналитика с событийной схемой (Я.Метрика + GA4 + кастом-события `view_item`, `add_to_cart`, `lead_submit`, `configurator_use`); **лог внутреннего поиска** → живая подпитка семантического ядра; данные B2B-заявок → будущая CRM/дэшборд.

**CI/CD:** Vercel + `tsc`/`eslint` + **Lighthouse CI с бюджетом Core Web Vitals** (деградация скорости не проходит в прод — привязано к SEO-цели).

## 15. Метрики успеха

- **B2C:** конверсия в заказ, AOV, доля Kaspi-оплат.
- **B2B:** заявок/мес, стоимость лида, конверсия калькулятора в заявку.
- **SEO (приоритет):** позиции по ядру, органический трафик, рост доли Яндекс, индексация лендингов, CTR из сниппетов.

## 16. Открытые вопросы и риски

1. Нет прямой цифры объёма рынка караоке-оборудования в KZ — макропрокси не заменяют её.
2. Неизвестна реальная доля продаж через Kaspi vs прямые, и влияние рассрочки на конверсию дорогих систем.
3. Факторы решения покупателя выведены из позиционирования вендоров, не из первичных опросов.
4. Сезонность и точные частоты запросов — нужен Wordstat/Keyword Planner (Фаза 0).
5. Финансовые данные Kaspi — FY2024/Q2 2024 (на ~16–24 мес устарели), номинальные в тенге.
6. CMS-выбор (Sanity/Strapi) не финализирован — зависит от требования резидентности данных.
