# design-docs — справочник по визуализации

Единый набор доков по цвету и визуализации: правило, справочник, готовая палитра, чекер и паст-ин.
Собрано 2026-07-07. Цвета/шрифты/бренд подбираются локально в каждом проекте — доки задают роли, ограничения и процесс.

## Что где

| Файл | Что это | Когда открывать |
|---|---|---|
| **[VISUALIZATION.md](VISUALIZATION.md)** | **Главное.** Переносимый стандарт визуализации/UI: токен-контракт, дисциплина нейтралей, контраст WCAG, тёмная тема, типографика/сетка, анти-ИИ, процесс, DoD. Внутри — готовая JS-функция контраста. | Перед любой визуализацией/UI. Раздавать проектам. |
| [color-systems-guide.md](color-systems-guide.md) | Справочник: проверенные принципы, established-системы (Material/Apple/Carbon/Tailwind/Radix/Stripe), готовые палитры с hex, источники. | Когда нужен пример/референс или выбор системы. |
| [palette-medlog.md](palette-medlog.md) | Готовая палитра MedLog под пациента (light+dark токены, контраст, семантика статусов). | Работа над MedLog или как образец «Stripe → ре-тюнинг». |
| [check.mjs](check.mjs) | Контраст-чекер WCAG 2.1. Впиши пары → `node check.mjs`. | Проверить любую пару текст/фон. |
| [session-handoff.md](session-handoff.md) | Готовый паст-ин: даёшь в новой сессии проекта — подключит стандарт. | Завести стандарт в новом проекте. |

## Быстрый старт

1. Новая визуализация → открой **VISUALIZATION.md**, следуй токен-контракту и DoD.
2. Нужен пример палитры → **color-systems-guide.md** или **palette-medlog.md**.
3. Проверить контраст → впиши пары в **check.mjs**, `node check.mjs`.
4. Подключить к новому проекту → скопируй блок из **session-handoff.md** в новую сессию.

## Обязательный минимум (из стандарта)

- Цвета — роль-токенами `--color-*`, не хардкод hex.
- light + dark (dark — ре-тюнинг, не инверсия).
- Каждая пара текст/фон ≥ 4.5:1 (тело) / ≥ 3:1 (крупное/UI) — реальными числами.
- Семантика состояний — отдельные роли, не бренд-акцент.
- Ноль пунктов из «Анти-ИИ».
