# VISUALIZATION.md — правило визуализации (переносимое)

> Универсальный, **цвето-агностичный** стандарт для любой визуализации и UI: артефакты, виджеты, мокапы, лендинги, темы продукта.
> Конкретные цвета, шрифты и бренд подбираются **локально в каждом проекте** — здесь заданы только роли, ограничения, процесс и «нельзя».
> Порядок приоритета: **слова пользователя → существующая дизайн-система проекта → это правило → дефолты**.
> Как применять: скопировать в `CLAUDE.md` проекта или держать как `docs/VISUALIZATION.md`.

---

## 0. Definition of Done (обязательный минимум)

Визуализация готова, только если:
- [ ] Цвета заданы **ролями-токенами**, а не хардкодом hex по месту.
- [ ] Есть **light и dark** (dark — ре-тюнинг, не инверсия).
- [ ] **Каждая** пара текст/фон проверена по WCAG: тело ≥ 4.5:1, крупное/UI ≥ 3:1. Числа реальные (посчитаны), не «на глаз».
- [ ] Семантика состояний (success/danger/warning/info) — **отдельные роли**, не бренд-акцент.
- [ ] Нет ни одного пункта из раздела 6 («Анти-ИИ»).
- [ ] Нет горизонтального скролла страницы; фокус-состояния видимы; `prefers-reduced-motion` учтён.

---

## 1. Токен-контракт (роли, не цвета)

Токен именует **применение**; значение подставляет тема. UI ре-темизируется без правки кода. Нейминг — `--color-*`.

```css
:root{
  /* Поверхности — от светлого к тёмному (light) */
  --color-page:      /* фон приложения (60%) */;
  --color-bg:        /* карточка / панель (30%) */;
  --color-surface:   /* чипы, вложенные подложки */;
  /* Текст */
  --color-fg:        /* основной текст · ≥ 4.5:1 на bg (цель AAA 7:1) */;
  --color-muted-fg:  /* вторичный текст · ≥ 4.5:1 на bg */;
  /* Линии */
  --color-border:    /* hairline · ≥ 3:1 если несёт смысл (границы инпутов) */;
  /* Бренд / действие (10%) */
  --color-primary:     /* ОДИН акцент */;
  --color-primary-fg:  /* текст на primary · ≥ 4.5:1 на primary */;
  --color-link:        /* ссылки · ≥ 4.5:1 на bg */;
  /* Семантика состояний — НЕ бренд-акцент */
  --color-success:  ; --color-danger:  ;
  --color-warning:  ; --color-warning-fg: /* тёмный текст на жёлтой заливке */;
  --color-info:     ;
}
.dark{
  /* Те же роли, ре-деривированы под тёмную поверхность (НЕ инверсия) */
  --color-page:    /* НЕ #000 — приподнятый тёмный */;
  --color-bg: ; --color-surface: ;
  --color-fg:      /* НЕ #fff — off-white */;
  --color-muted-fg: ; --color-border: ;
  --color-primary: /* светлее и десатурированнее light-версии */;
  --color-primary-fg: ; --color-link: ;
  --color-success: ; --color-danger: ; --color-warning: ; --color-warning-fg: ; --color-info: ;
}
```

**Правила текста в теме:** темизируемый текст — только `--color-fg` / `--color-muted-fg`. Никаких `dark:text-[#…]` / `dark:text-white` по месту — это возвращает whack-a-mole тёмной темы. On-primary текст — `--color-primary-fg`, не `#fff` (в тёмной primary может быть светлым).

## 2. Дисциплина нейтралей

Нейтрали (page/bg/surface/fg/muted/border) — 90% интерфейса. Они делают или ломают «профессиональность».

- **Выбирай температуру осознанно.** True-neutral, cool (синий подтон) или warm (янтарный подтон) — это ВЫБОР, а не случайность. Чистый `#808080` читается как «не думали».
- **Один подтон на всю шкалу.** Не мешай cool-фон с warm-текстом. Если тонируешь нейтрали в сторону акцента — делай это осознанно и слабо; по умолчанию держи нейтрали **нейтральными**, а насыщенность отдавай только акценту.
- **Насыщенность к краям.** В тёмных/светлых концах шкалы saturation визуально падает — чуть поднимай её на крайних шагах, иначе тинты выглядят выцветшими.
- **8–10 оттенков серого** нужно больше, чем кажется: текст, фоны, панели, контролы, disabled, границы.

## 3. Контраст и доступность (твёрдые пороги)

| Что | AA | AAA |
|---|---|---|
| Обычный текст (< 24px / < 18.5px bold) | **4.5:1** | 7:1 |
| Крупный текст (≥ 24px или ≥ 18.5px bold) | **3:1** | 4.5:1 |
| UI-элементы, границы, иконки, фокус | **3:1** | — |

Формула (проверяй **каждую** пару, не округляй — 4.499 не проходит 4.5):
```
contrast = (L1 + 0.05) / (L2 + 0.05)          // L1 — светлейший
L = 0.2126·R + 0.7152·G + 0.0722·B            // по линеаризованным sRGB-каналам
channel: c ≤ 0.03928 ? c/12.92 : ((c+0.055)/1.055)^2.4
```
- **Никогда не цветом единым** — дублируй смысл иконкой/подчёркиванием/текстом. Тест: переведи в grayscale — если состояние/ссылка/ошибка исчезли, цвет делает слишком много.
- **Фокус** — видимый outline (толщина + ≥ 3:1), не только смена hue.
- **APCA** (черновик WCAG 3) — sanity-check тёмной темы: WCAG 2 переоценивает контраст у near-black.

## 4. Тёмная тема — ре-тюнинг, не инверсия

- База — **приподнятый тёмный, не чистый чёрный** (эталон ≈ `#121212`): остаётся запас на elevation.
- Текст — **off-white, не чистый белый** (halation/свечение утомляет); либо белый с opacity-ярусами (high 87% / medium 60% / disabled 38%).
- **Десатурируй акценты** (−20…30% saturation, светлее), иначе цвет «вибрирует» на тёмном.
- Глубину задавай **светлыми surface-оверлеями**, а не тенями (тени на тёмном не читаются).
- Каждую роль **ре-деривируй** под тёмную поверхность; не инвертируй палитру механически.

---

## 5. Типографика · сетка · отступы

**Типографика**
- Пара осознанно: display (характерный, дозированно) + body (читаемый) + mono (данные/hex/код).
- Единая type-scale; заголовки — `text-wrap: balance`; тело ~ 65ch ширины строки.
- Капс-лейблы — с letter-spacing; колонки цифр — `font-variant-numeric: tabular-nums`.
- **В песочнице (артефакты/CSP)** не подключай шрифт с CDN — молчаливый фолбэк. Либо inline `@font-face` data-URI, либо честный system-stack.

**Сетка и отступы**
- Раскладку соседних групп делай `flex`/`grid` + `gap`, не per-element margin (схлопывается/дублируется).
- Широкий контент (таблицы, код, диаграммы) — в контейнере с `overflow-x:auto`; **тело страницы не скроллит вбок**.
- Радиусы и отступы — из фиксированной шкалы, не произвольные.

## 6. Анти-ИИ (чего НЕ делать)

Generic-визуал узнаётся мгновенно. Избегай:
1. **Чистый `#fff` / `#000`** фон/текст — бери off-white и «почти-чёрный» с лёгким подтоном.
2. **Дефолтный bootstrap-синий / unstyled shadcn-slate** — выбери отличительный бренд-hue.
3. **Тинты через `opacity`** вместо настоящей шкалы оттенков — на пёстром фоне грязнится.
4. **Тень-«клей»** под каждой карточкой — поверхности разделяй цветом + hairline; тень только у плавающего (оверлей, sticky, FAB, drawer).
5. **Радуга акцентов** — один доминантный акцент; остальное семантика точечно.
6. **Фиолет→синий градиент-hero на белом** — самый заезженный ИИ-приём.
7. **Ряды pill-чипов, CountUp-счётчики, эмодзи-маркеры секций, всё по центру, `rounded-*` везде** — маркеры «сгенерировано».
8. **Заголовки-«иконка+текст» сеткой на каждый блок**; нумерация 01/02 — только если контент реально последовательность.
9. **Дизайн без привязки к теме** — черпай отличительность из мира предмета (материалы, лексика, инструменты).

---

## 7. Процесс подбора палитры (с нуля, локально)

1. **Один бренд-hue → шкала.** База (≈ шаг 500 / Radix-9) должна годиться фоном кнопки; выведи 10 шагов `50→950`, тёмный — текст, светлый — тинты.
2. **Нейтральная шкала** (8–10) с осознанной температурой (раздел 2).
3. **Назначь роли** по токен-контракту (раздел 1); разложи по 60-30-10.
4. **Семантика состояний** — отдельные success/danger/warning/info, не бренд-акцент.
5. **Проверь контраст** каждой пары (раздел 3). Правь hex, пока не пройдёт AA.
6. **Ре-тюнингуй dark** (раздел 4) и **перепроверь контраст ещё раз**.

Опорные системы (что подсмотреть): Material 3 (`on-*`-пары, surface-тиры), Apple HIG (семантика label/background), IBM Carbon (gray + один accent), Tailwind (шаги 50→950), Radix (12-шаговый контракт ролей), Stripe (перцептивная равномерность CIELAB).

## 8. Верификация перед сдачей

- Посчитай контраст **каждой** отображаемой/используемой пары (скрипт/функция, не глаз). Провал ниже порога — правь, не показывай.
- Отчитывайся честными числами; если пара проходит только как «AA Large» — так и помечай.
- Прогони чек-лист из раздела 0.

### Готовая функция контраста (WCAG 2.1)

Вставь в `check.mjs` рядом с проектом, впиши свои пары `[fg, bg, label]`, запусти `node check.mjs`. Те же функции работают и в браузере (для живых бейджей). Разбирает 3- и 6-значный hex.

```js
// WCAG 2.1 contrast — проверка пар текст/фон перед сдачей.
function _lin(c){ c/=255; return c<=0.03928 ? c/12.92 : Math.pow((c+0.055)/1.055, 2.4); }

function luminance(hex){
  hex = hex.replace('#','');
  if (hex.length === 3) hex = hex.split('').map(x => x+x).join('');   // #abc -> #aabbcc
  const r = parseInt(hex.slice(0,2),16), g = parseInt(hex.slice(2,4),16), b = parseInt(hex.slice(4,6),16);
  return 0.2126*_lin(r) + 0.7152*_lin(g) + 0.0722*_lin(b);
}

function contrast(a, b){
  const l1 = luminance(a), l2 = luminance(b);
  const hi = Math.max(l1,l2), lo = Math.min(l1,l2);
  return (hi + 0.05) / (lo + 0.05);
}

// 'AA' ≥4.5 (тело) · 'AA Large' ≥3 (крупное/UI) · 'FAIL' <3
function grade(r){ return r >= 4.5 ? 'AA' : r >= 3 ? 'AA Large' : 'FAIL'; }

// Прогон списка пар [fg, bg, label] → печать + true, если всё ≥3:1
function checkPairs(pairs){
  let fails = 0;
  for (const [fg, bg, label] of pairs){
    const r = contrast(fg, bg), g = grade(r);
    console.log(`  ${g.padEnd(9)} ${r.toFixed(2).padStart(6)}:1  ${label||''}  (${fg} on ${bg})`);
    if (g === 'FAIL') fails++;
  }
  console.log(fails ? `\n✗ ${fails} провал(ов) ниже 3:1` : '\n✓ все пары ≥ 3:1');
  return fails === 0;
}

// --- впиши свои пары и запусти ---
checkPairs([
  ['#16202f', '#ffffff', 'текст/карта'],
  ['#57617a', '#ffffff', 'muted/карта'],
  ['#ffffff', '#0d7a6e', 'on-primary'],
]);
```
