# Паст-ин для новой сессии проекта

> Скопируй блок ниже и дай его в новой сессии любого проекта — она подключит стандарт визуализации.
> Универсальный: имя проекта нигде не захардкожено (сессия определит по каталогу).

```text
Задача: подключить единый стандарт визуализации к этому проекту.

1. Прочитай файл ~/Desktop/design-docs/VISUALIZATION.md — это переносимый
   цвето-агностичный стандарт визуализации/UI (роли-токены, контраст WCAG,
   light/dark, семантика состояний, анти-ИИ). Конкретные цвета/шрифты/бренд
   подбираются ЛОКАЛЬНО в проекте. Ничего в стандарте не меняй.
   (Если файл недоступен — попроси меня вставить его содержимое.)

2. Скопируй его в репозиторий как docs/VISUALIZATION.md (если ещё нет).
   Готовый контраст-чекер — ~/Desktop/design-docs/check.mjs; при желании
   положи его в проект тоже.

3. Впиши в CLAUDE.md проекта, в раздел про дизайн/UI-конвенции, строку-ссылку:
   «Визуализация и цвет — по стандарту docs/VISUALIZATION.md
   (роли-токены --color-*, контраст WCAG AA, light/dark, семантика, анти-ИИ).»

4. Сверь текущую тему/токены проекта со стандартом. Если есть расхождения
   (хардкод hex вместо ролей-токенов, нет dark-темы, непроверенный контраст,
   тень-«клей», чистый #fff/#000, дефолт-синий) — НЕ чини молча: выпиши их
   списком в docs/HANDOFF.md и покажи мне перед правкой.

5. Обнови доки и закоммить: если в проекте есть правило «доки+коммит+пуш» —
   следуй ему (доки+код одним коммитом → push сразу → проверить CI);
   иначе просто обнови HANDOFF, сделай один коммит и запушь.

Definition of Done для ЛЮБОЙ новой визуализации в этом проекте:
- цвета — роль-токенами --color-*, не хардкод hex по месту;
- есть light и dark (dark — ре-тюнинг, не инверсия: не #000 базой, не #fff текстом,
  акценты десатурированы);
- каждая пара текст/фон проверена по WCAG (тело ≥ 4.5:1, крупное/UI ≥ 3:1) —
  реальными числами, не на глаз;
- семантика состояний (success/danger/warning/info) — отдельные роли, не бренд-акцент;
- ноль пунктов из раздела «Анти-ИИ» стандарта.
```

**Передаёшь на другую машину?** Файлы `~/Desktop/design-docs/` там могут быть недоступны — тогда приложи их содержимое к сессии или попроси вставить.
