# 📘 Как править сайт самой (без программистов и ИИ)

Эта инструкция — для новичка. Никакого кода знать не нужно. Всё правится в браузере на сайте GitHub, а сайт обновляется сам.

---

## Как это всё работает (в двух словах)

1. Весь «контент» сайта (товары, цены, тексты, телефон, статьи) лежит в нескольких **текстовых файлах** на GitHub.
2. Ты открываешь файл прямо в браузере, меняешь нужное, нажимаешь **«Сохранить» (Commit)**.
3. Через **1–2 минуты сайт сам обновляется** — Cloudflare пересобирает его автоматически.

Ничего устанавливать не надо. Главное — аккуратность (не удалять кавычки и запятые).

> **Обновление (Фаза 1, июль 2026):** раньше весь контент лежал в одном файле `site.ts` вперемешку с кодом программиста. Теперь контент **переехал в отдельную папку `apps/web/content/`** — там лежат обычные JSON-файлы, по одному на каждый вид контента (товары, блог, контакты и т.д.), и кода там больше нет. Править стало безопаснее: ничего программистского вокруг текста, ниже риск случайно что-то сломать. Принцип работы (Шаг 1 — «карандаш → правка → Commit») не изменился, изменились только **пути к файлам** — см. таблицу ниже.

## Шаг 1. Как вообще отредактировать файл на GitHub (общий приём)

Запомни один раз — дальше всё по этой схеме:

1. Открой нужный файл на github.com.
2. Нажми значок **карандаша** ✏️ (вверху справа, «Edit this file»).
3. Внеси изменения.
4. Прокрути вниз → зелёная кнопка **«Commit changes»**.
5. В окошке напиши коротко, что менял (напр. «цена AST-250») → ещё раз **«Commit changes»**.
6. Подожди 1–2 минуты → обнови сайт и проверь.

Готово. Откат, если что — в Шаге 9.

## 🔎 Как точечно найти, где менять (через браузер, как «режим разработчика»)

Два инструмента, оба в браузере, ничего ставить не надо.

**1. «Просмотреть код» (Inspect / F12) — взять точный текст с сайта.**
- На сайте наведись на нужный элемент → правый клик → **«Просмотреть код»** (или клавиша F12).
- Скопируй точный кусок текста, который видишь (название «RCF EVOX 8», абзац статьи и т.п.).
- ⚠️ Цены ищи **по названию товара, а не по числу**: на сайте «1 500 000 ₸», а в файле это `1500000` — по «1 500 000» не найдётся.

**2. github.dev — редактор как у разработчика, прямо в браузере.**
- Открой репозиторий на github.com и нажми клавишу **`.`** (точка). Откроется VS Code в браузере (адрес станет github.dev).
- Поиск по всему проекту: **Ctrl+Shift+F** (на Mac — **Cmd+Shift+F**).
- Вставь скопированный текст → редактор покажет нужный **файл и строку**. Кликни — попадёшь точно в место.
- Поменяй → слева значок «Source Control» (ветка) → напиши, что менял → **Commit & Push**.
- Через 1–2 минуты сайт обновится.

**Связка целиком (точечно):**
1. Видишь на сайте, что поменять.
2. Правый клик → «Просмотреть код» → скопируй точный текст.
3. В репозитории нажми `.` → откроется github.dev.
4. Ctrl/Cmd+Shift+F → вставь текст → перейди в найденную строку.
5. Исправь → Commit & Push.

> Не хочешь редактор? Ищи прямо на github.com — кнопка поиска вверху, вставляешь текст, «In this repository». Найдёт файл, дальше правишь карандашом (как в Шаге 1).

## Шаг 2. Поменять ЦЕНУ товара

Открой `apps/web/content/products.json`. Найди нужный товар — каждая запись выглядит так:

```json
{
  "slug": "ast-250",
  "type": "sistema",
  "model": "AST-250",
  "brand": "AST",
  "price": 1500000,
  "priceOld": 2664000,
  "rating": 4.9,
  "reviewsCount": 23,
  "inStock": true,
  "image": "/products/ast-250.webp"
},
```

Найди `"price": 1500000` и поменяй число.
- ✅ Правильно: `"price": 1480000`
- ❌ Нельзя: пробелы, ₸, запятые внутри числа (`"price": 1 480 000` или `"price": "1480000₸"` — сломает).
- Старая (зачёркнутая) цена, если есть скидка, — поле `priceOld`, тем же способом.

Сохрани (Шаг 1). Цена обновится и в каталоге, и на карточке, и в сравнении.

## Шаг 3. Поменять НАЗВАНИЕ или НАЛИЧИЕ

В той же записи товара в `products.json`:
- Название: `"model": "AST-250"` → меняешь текст **внутри кавычек**: `"model": "AST-260"`.
- Бренд: `"brand": "AST"`.
- Наличие: `"inStock": true` → `"inStock": false` (станет «под заказ»).

> Текст всегда внутри прямых кавычек `"..."`. Числа (цена, рейтинг) — без кавычек. В JSON-файлах в кавычках также и **ключи** (слово слева от двоеточия, напр. `"price"`) — их не трогай, меняй только то, что справа от `:`.

## Шаг 4. Добавить НОВЫЙ товар

Проще всего — **скопировать готовую запись** и поменять в ней значения.

1. Выдели целиком одну запись товара (от `{` до `},`) в `apps/web/content/products.json`.
2. Скопируй, вставь рядом новой записью.
3. Поменяй в копии:
   - `slug` — короткое имя латиницей без пробелов, **уникальное** (напр. `"yamaha-mg10"`). Это адрес товара.
   - `type` — один из: `"sistema"`, `"akustika"`, `"mikrofon"`, `"sub"`, `"miksher"`.
   - `model`, `brand`, `price`, при желании `kit` (список того, что в комплекте), `image`.

Пример новой записи акустики:
```json
{ "slug": "yamaha-dbr12", "type": "akustika", "model": "Yamaha DBR12", "brand": "Yamaha", "price": 350000, "inStock": true },
```

> Каждая запись обязательно заканчивается запятой `,` (кроме самой последней в файле). Не потеряй её.

## Шаг 5. Добавить или сменить ФОТО товара

Два способа:

**А. Свой файл (для реальных фото):**
1. Открой папку `apps/web/public/products/` на GitHub → **Add file → Upload files** → загрузи картинку.
   Назови файл по slug товара, напр. `rcf-evox-8.jpg`.
2. В `apps/web/content/products.json` у товара добавь/поменяй: `"image": "/products/rcf-evox-8.jpg"`.

**Б. Ссылка на картинку из интернета:**
```json
"image": "https://адрес-картинки.jpg"
```

Если поля `image` нет — показывается иконка-заглушка (это нормально).
Рекомендация по фото: ~800×600 пикселей и больше, до ~300 КБ (сжать можно на squoosh.app).

## Шаг 6. Поменять цены в КАЛЬКУЛЯТОРЕ сметы

Калькулятор сметы — единственное, что **не переехало** в `content/`: это отдельная логика, файл `apps/web/src/lib/components.ts`. Меняй числа в `price:`:
```
mic: { id: "mic", name: "Радиомикрофон Shure BLX24/SM58", price: 60000 },
```
→ поменяй `price: 60000` на нужное. Так же — у акустики (`acoustics`) и базовых систем (`baseSystems`).

Открой `apps/web/content/site-config.json` — там весь блок контактов:
```json

"whatsapp": "77075799995",
"address": "Алматы, ул. Муканова, офис 8",
"hours": "Пн–Сб · 10:00–19:00",

```
(`whatsapp` — только цифры, для кнопки WhatsApp.) Меняешь текст внутри кавычек. Адрес обновится и на странице «Контакты», и на карте.

## Шаг 8. Тексты: блог, кейсы, песни

Каждый вид текста — в своём файле в `apps/web/content/`:
- **Блог** — `blog.json`. У статьи: `title` (заголовок), `excerpt` (короткое описание), `body` (абзацы — каждый абзац отдельной строкой в кавычках через запятую, в квадратных скобках `[ ]`), `faq` (вопрос-ответ).
- **Истории клиентов (отзывы)** — `stories.json`: `author`, `venue`, `text` (текст отзыва), `tags`.
- **Кейсы** — `cases.json`: `venue` (заведение), `city`, `area` (площадь), `system`, `quote` (отзыв), `author`.
- **Песни** — `songs.json`: `title`, `artist`, `lang` (`"KZ"`, `"RU"`, `"EN"`).
- **Заголовки/описания страниц (SEO)** — `page-meta.json`, `scenarios.json`, `brands.json`.

Принцип тот же: меняешь текст в кавычках или копируешь запись для новой.

## Шаг 9. ⚠️ Главные правила, чтобы не сломать

- **Не удаляй** кавычки `"`, запятые `,`, скобки `{ }` и `[ ]`. Меняй только текст/числа.
- Кавычки только **прямые** `"..."` (не «ёлочки» и не „кавычки" из Word).
- Текст — внутри кавычек. Числа (цена, рейтинг) — без кавычек. Ключи (слово перед двоеточием, напр. `"price"`) тоже в кавычках — не трогай их, только значение справа от `:`.
- Каждая запись-товар/запись внутри списка заканчивается запятой `,`.
- **Важно для JSON-файлов (`content/*.json`):** после **самой последней** записи в списке `[ ]` или последнего поля в `{ }` запятой быть **не должно** — иначе файл не откроется (упадёт сборка). Если правишь запись не последнюю — как обычно, запятая нужна.
- Если внутри текста нужны кавычки — лучше избегай, или напиши без них.

## Шаг 10. Если что-то сломалось — как откатить

1. После сохранения GitHub запускает проверку. Зайди во вкладку **Actions** — если стоит красный крестик ❌, твоя правка сломала сборку (сайт НЕ обновится старый останется живым — не страшно).
2. **Откат к рабочей версии:** вкладка **Commits** (история) → найди свою правку → кнопка **«Revert»** (отменить) → Commit. Сайт вернётся как было.

Откатить можно всегда — старые версии не теряются.
