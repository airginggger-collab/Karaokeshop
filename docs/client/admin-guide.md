# Личный кабинет — как заходить и работать

> Источник для страницы сайта **`/admin-guide/`** (собирается в Фазе 2 вместе с рабочим кабинетом `/admin`). Пока визуальный редактор в разработке — актуальный способ правок описан в конце, в разделе «Пока кабинет не готов».

Личный кабинет — это визуальный редактор сайта. Ты меняешь товары, цены, тексты, фото и статьи **через удобные формы**, без кода. После сохранения сайт сам обновляется за 1–2 минуты.

---

## Как зайти

1. Открой в браузере адрес: **`karaokeshop.kz/admin`** (или адрес прода `…workers.dev/admin`).
2. Нажми **«Войти через GitHub»**.
3. Введи логин и пароль своего GitHub-аккаунта (того, что привязан к сайту). Первый раз GitHub спросит разрешение — нажми **Authorize**.
4. Готово — откроется кабинет со списком разделов слева.

> Вход только через GitHub — это и есть твой ключ. Пароль от GitHub никому не давай. Если входишь с нового устройства, GitHub может прислать код на почту — это нормально.

---

## Что где (разделы слева)

| Раздел | Что редактируешь |
|---|---|
| **Товары** | Каталог: название, цена, скидка, тип, фото, «в наличии», описание, комплектация |
| **Контакты и сайт** | Телефон, WhatsApp, адрес, почта, часы работы, заголовки/описания страниц (SEO) |
| **Блог** | Статьи: заголовок, текст, вопросы-ответы |
| **Истории и кейсы** | Отзывы-истории установок, кейсы объектов |
| **Лендинги и бренды** | Заголовки и описания страниц «для дома / для бизнеса / бренды» и т.п. |

---

## Как поменять цену товара (пример)

1. Слева нажми **Товары**.
2. Выбери товар в списке (например, «AST-250»).
3. Найди поле **Цена**, впиши новое число (только цифры, без пробелов и «₸»).
4. Нажми **Сохранить** (Save) вверху справа.
5. Через 1–2 минуты новая цена уже на сайте.

Так же меняются любые поля: название, «в наличии» (галочка), скидка, описание.

---

## Как загрузить фото товара

1. Открой товар → поле **Фото**.
2. Нажми **Выбрать изображение** → **Загрузить** и выбери файл с компьютера.
3. Сохрани. Фото появится на карточке товара и на его странице.

> Лучше квадратное или горизонтальное фото, чёткое, на однотонном фоне. Слишком тяжёлые файлы (больше ~2 МБ) перед загрузкой лучше сжать.

---

## Как добавить статью в блог

1. Раздел **Блог** → **Создать запись** (New).
2. Заполни: **Заголовок**, короткое **Описание** (для превью), **Текст** (по абзацам), при желании — **Вопросы-ответы**.
3. Сохрани. Новая статья появится в разделе «Блог» на сайте.

---

## Что происходит после «Сохранить»

Кабинет сохраняет правку в GitHub (там хранится весь сайт), и сайт автоматически пересобирается. **Ждать ~1–2 минуты** — потом обнови страницу сайта, изменения будут видны.

- **Каждая правка сохраняется в историю** — если что-то пошло не так, прежнюю версию всегда можно вернуть (скажи об этом тому, кто ведёт сайт).
- **Сломать сайт формами почти невозможно** — цена принимает только числа, обязательные поля нельзя оставить пустыми.

---

## Чего не стоит делать

- Не оставляй **обязательные поля** (название, цена) пустыми.
- Не вставляй в цену буквы, пробелы, «₸» — только число.
- Не удаляй товар, если не уверена — лучше сними галочку «в наличии».

---

## Если правка не появилась

1. Прошло меньше 2 минут — просто подожди и обнови страницу (Ctrl/Cmd + R).
2. Прошло больше 5 минут — возможно, сборка сайта не прошла. Напиши тому, кто ведёт сайт: он проверит.

---

## Пока кабинет не готов (Фаза 1)

Визуальный редактор `/admin` появится в Фазе 2. Сейчас весь контент уже вынесен в **понятные файлы** в папке `apps/web/content/` на GitHub (`products.json` — товары, `blog.json` — статьи, `site-config.json` — контакты и т.д.). Их можно править прямо на GitHub в браузере: аккуратно, не удаляя кавычки и запятые (это безопаснее, чем прежняя правка кода). Подробная пошаговая инструкция по GitHub-правке — [`docs/redaktirovanie-sajta.md`](../redaktirovanie-sajta.md).
