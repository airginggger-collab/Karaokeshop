# Аудит сайта karaokeshop — 2026-06-22

> Глубокий многомерный аудит живого сайта `https://karaokeshop.airg-inggger.workers.dev` и репозитория.
> Метод: разбор живого сайта (desktop + mobile скриншоты, HTML/SEO через curl) + ревизия репозитория (read-only).
> Рабочий план по устранению — в [`fix-plan-2026-06-22.md`](fix-plan-2026-06-22.md).

## Резюме

Сильная инженерия и аккуратная структура, но сайт «пустой внутри»: фейковые сток-фото, тонкие карточки товара, и весь SEO-слой указывает на ещё не подключённый домен. Это хорошо построенная оболочка, ждущая реального контента и запуска.

## 1. Визуал / дизайн — B

**Сильно:** чистый shadcn-минимал, единый зелёный акцент, нейтральные поверхности, скругления, хорошие отступы и типографика; крепкий адаптив (mobile home/catalog собраны корректно); тёмные сегмент-блоки дают контраст.

**Проблемы:**
- 🔴 **Все фото товаров — сток с Unsplash, не относящийся к продукту.** Карточка AST-250 (система за 1.5М ₸) — фото спальни; в каталоге концертные толпы, наушники, бары. Рушит визуальное доверие сильнее всего. (`apps/web/src/lib/site.ts` — поля `image:` с `images.unsplash.com`)
- 🟡 Карточки каталога держатся на «настроенческих» фото, а не на ясности продукта.
- 🟡 Есть переключатель темы (☀) — проверить, что тёмная не ломается.

## 2. UX / UI — B

**Сильно:**
- Понятная сценарная навигация (Каталог / Комплекты / Калькулятор / Под ключ / Песни).
- ⭐ **Калькулятор сметы — лучший элемент сайта:** интерактивный (тип зала, площадь, микрофоны, опции), живой итог с разбивкой, CTA «Заявка со сметой в WhatsApp». Прямо отрабатывает позиционирование «караоке без ошибки в выборе».
- Рабочие фильтры в каталоге, хлебные крошки, блок «Как мы работаем», отзывы.

**Проблемы:**
- 🔴 **Страница товара крайне тонкая.** Для товара за 1.5М ₸ на ней только фото + заголовок + цена + 3 CTA + «Что в комплекте» чипсами. Нет: таблицы характеристик, описания, галереи, текста отзывов, условий доставки/гарантии, похожих товаров, FAQ. (`apps/web/src/app/product/[slug]/page.tsx`)
- 🟡 «4.9 · 23 отзыва» — но ни одного отзыва не показано → необоснованное доверие.
- 🟡 Кнопка «В корзину» для монтируемой системы за 1.5М ₸ спорна — реальная конверсия идёт через консультацию/WhatsApp; корзина добавляет трения.
- ⚪ FOUC: плавающая иконка WhatsApp (`h-7 w-7` без `width/height`) и навигация кратко мигают неоформленными на медленной загрузке (реальным пользователям почти незаметно).

## 3. Продажи / конверсия — C+

**Сильно:** мощный оффер «Караоке без ошибки в выборе»; много путей конверсии (калькулятор→WhatsApp, консультация, телефон, корзина); доверие (с 2012, офиц. дилер, сервис-центр в Алматы, реальные контакты); рассрочка Kaspi 0-0-12 и цена «от X ₸/мес»; сегментация дом/бизнес.

**Проблемы:**
- 🔴 Сток-фото убивают доверие там, где решаются на дорогую покупку.
- 🔴 Тонкие карточки товара = низкая конверсия на «денежных» страницах.
- 🟡 **Соцдоказательство неглубокое:** счётчик отзывов есть, контента нет; нет фото реальных монтажей, логотипов заведений-клиентов.
- 🟡 **Дрейф контента:** Kaspi/рассрочка живёт на сайте, хотя коммиты + CLAUDE.md заявляют «Kaspi удалён» (≈21 вхождение в `lib/site.ts`, `components/Hero.tsx`, `components/CalculatorClient.tsx`, `app/komplekty/[area]/page.tsx`).
- 🟡 Нет OG-тегов → ссылки в WhatsApp/Telegram (главные каналы KZ) выглядят голым текстом.

## 4. Технический / SEO — C+

- 🔴 **Весь canonical/sitemap/JSON-LD указывает на `www.karaokeshop.kz`, а живой хост — `*.workers.dev`.** Сайт фактически сам себя деиндексирует, пока домен не подключён.
- 🔴 **На главной `/` нет canonical** (на всех остальных есть).
- 🔴 **`/dlya-doma` и `/karaoke/dlya-doma`** — обе 200, само-canonical, почти одинаковые → каннибализация ключей.
- 🟡 Product-schema без `image` и без `BreadcrumbList`; нет OG/Twitter; JS ~483 КБ (риск для Lighthouse perf ≥0.90).
- ✅ Хорошо: уникальные title/description, по одному `<h1>`, валидные Product/Offer + LocalBusiness, alt + lazy на всех `<img>`, реальные контакты, рабочий 404, корректные редиректы; `/sravnit` правильно `noindex`.
- ⚠️ Lighthouse CI (`lighthouserc.json`) аудитит только 3 URL (`/index.html`, `/karaoke/dlya-bara.html`, `/product/ast-250.html`) — узкое покрытие.

## 5. Порядок в репозитории — B+

**Закоммиченное дерево дисциплинировано** (строгий TS, чистые barrels `@kk/*`, нет артефактов в гите, реальный CI + Lighthouse-бюджеты, ко-локация тестов). Минусы — в рабочей директории и дрейфе:
- 🔴 **768 МБ ZIP** (`Для авторизованных дилеров…zip`) и **35 МБ папка `скриныыы караокешоп/`** в корне репо (untracked, но `*.zip` не в `.gitignore`) — один `git add .` от катастрофы.
- 🟡 **Дублирующий источник токенов:** корневой `tokens/` (DTCG) — орфан; сборка (`packages/tokens/build.mjs`) читает только `packages/tokens/tokens.json`.
- 🟡 3 мёртвых компонента: `apps/web/src/components/{FilterRail,SectionHeading,TrustStrip}.tsx`.
- 🟡 Нет ESLint/Prettier-конфигов; `packages/ui` lint — заглушка (`echo`); `engines` не пиннится; `*.tsbuildinfo` и `.claude/` не игнорятся; `.next/` в `.gitignore` дублируется.
