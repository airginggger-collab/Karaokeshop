# adr/ — Architecture Decision Records

Каждое значимое решение — отдельный файл `NNNN-<тема>.md`. Формат: контекст → решение → отвергнутые альтернативы → последствия.

**Запланированные ADR:**
- `0001-cms.md` — выбор CMS (Sanity vs Strapi). Критерии: резидентность данных в KZ, бюджет DevOps, скорость старта, i18n, история версий. *(Task 5 плана Фазы 0)*
- `0002-commerce.md` — слой коммерции (Medusa / Snipcart / commerce в CMS) + интеграция Kaspi.
- `0003-i18n.md` — стратегия RU/KZ (hreflang, структура URL, локализация контента).

**Уже принятые решения (зафиксированы в [стратегии](../strategy/2026-06-18-redesign-strategy.md)):**
- Архитектура — модель B (единая витрина + B2B-трек).
- Стек — Next.js (SSR/SSG) + Headless CMS, монорепо Turborepo.
