# ADR 0001 — Выбор CMS

**Статус:** Предложено (финал — после ответа заказчика по резидентности данных и бюджету, Фаза 0)
**Дата:** 2026-06-18

## Контекст
Нужна Headless CMS как админка: владелец/маркетолог редактирует товары, цены, комплекты, B2B-кейсы, каталог песен, блог, SEO-поля и RU/KZ-локали **без разработчика**. Требования: история версий контента (откат), i18n (RU сейчас, KZ позже), удобная Studio, интеграция с Next.js, разумная стоимость.

## Критерии и оценка

| Критерий | Sanity | Strapi |
|---|---|---|
| История версий / откат | ★★★ нативно, document history | ★★ (draft/publish; история — через плагины/v5) |
| i18n RU/KZ | ★★★ document-level i18n | ★★ плагин i18n |
| Скорость старта | ★★★ хостинг, ноль DevOps | ★★ нужен сервер/БД/деплой |
| Контроль и резидентность данных | ★ данные в облаке Sanity | ★★★ self-hosted, своя БД в KZ |
| Стоимость на старте | ★★★ щедрый free-tier | ★★ хостинг сервера/БД |
| Developer experience | ★★★ Portable Text, GROQ, real-time | ★★ REST/GraphQL, типизация |
| Сопровождение владельцем | ★★★ ничего не админить | ★ требует обновлений/бэкапов |

## Решение
**По умолчанию — Sanity.** Для портфолио-проекта без заявленного юридического требования резидентности данных в KZ Sanity даёт лучшую историю версий, i18n и нулевой DevOps — владелец ведёт контент, никто не админит сервер.

**Переключаемся на Strapi, если** заказчик подтвердит одно из:
- юридическое/политическое требование хранить данные в Казахстане;
- потребность в полном контроле БД и интеграции с локальной инфраструктурой (1С/склад) на уровне БД;
- неприемлемость SaaS-зависимости/ценообразования при росте.

## Последствия
- При Sanity: схема — в `packages/cms-schema` (Sanity schema), контент в Sanity Content Lake; на фронте GROQ-запросы из Next.js (ISR).
- При Strapi: поднимаем сервер + Postgres (резидентно), схема — content-types, REST/GraphQL.
- В обоих случаях модель контента ([data-model](../context/data-model.md)) одинакова — выбор CMS не меняет сущности, только реализацию. Это снижает стоимость смены решения.
