# ADR 0002 — Хостинг фронтенда

**Статус:** Принято → реализовано с отклонением (см. ниже)
**Дата:** 2026-06-18 | **Обновлено:** 2026-06-20

## Контекст
Философия «максимум бесплатно», разработка самостоятельная. Нужен хостинг под Next.js (SSR/SSG) с хорошей скоростью для SEO. Ограничение: **Vercel Hobby по ToS — только для некоммерции**, а это коммерческий магазин → бесплатный Vercel неприменим легально. Нужен бесплатный хостинг, разрешающий коммерцию, и при этом возможность переезда без переписывания.

## Решение (планировалось)
**Старт на Cloudflare Pages** (бесплатно, коммерция разрешена, безлимитный трафик, глобальный CDN — хорошо для Core Web Vitals и SEO). Деплой Next.js — через адаптер (`@cloudflare/next-on-pages` / OpenNext for Cloudflare).

## Как реализовано фактически

Вместо Cloudflare Pages использован **assets-only Worker** (`wrangler.toml`):
```toml
name = "karaokeshop"
compatibility_date = "2025-06-01"
[assets]
directory = "./apps/web/out"
not_found_handling = "404-page"
```
Live URL: `https://karaokeshop.airg-inggger.workers.dev/` (`*.workers.dev`, не `*.pages.dev`). Адаптер не нужен — сайт `output: "export"`, чистая статика. Авто-деплой через Cloudflare-интеграцию с GitHub при push в `main`.

Принцип переносимости сохранён: код host-agnostic, при переезде на Pages/Vercel — только смена `wrangler.toml`.

**Принцип переносимости:** код пишем host-agnostic, чтобы переезд на Vercel Pro был «сменой таргета деплоя», а не рефакторингом.

## Альтернативы

| Вариант | Плюсы | Минусы |
|---|---|---|
| **Cloudflare Pages** (выбран) | ₸0, коммерция OK, безлимит-трафик, топ-CDN | SSR через адаптер; edge-runtime ограничения (часть Node-API) |
| Netlify free | ₸0, коммерция OK, очень Next-нативный | лимит трафика 100 ГБ/мес и build-минут (overage платный) |
| Vercel Pro | нативный Next.js, ISR/Image «из коробки» | $20/мес/юзер — не вписывается в «₸0» |
| Vercel Hobby | нативный, ₸0 | **запрещён для коммерции** (ToS) |

## Условия переезда на Vercel Pro (триггеры)
- Упираемся в ограничения edge-runtime Cloudflare (нужны Node-API/тяжёлые серверные операции).
- Нужны нативные ISR / Next Image без возни с адаптерами.
- Команда/трафик выросли, $20/мес перестали быть проблемой.

## Последствия / как сохраняем переносимость
- Использовать **стандартные возможности Next.js**, избегать Vercel-only лок-ина.
- Ревалидацию (ISR) и оптимизацию изображений держать на host-agnostic паттернах (CDN-loader для картинок, явная стратегия revalidate).
- CI собирает один и тот же артефакт; деплой-таргет — переключаемая настройка.
- Резерв: Netlify free как запасной бесплатный вариант с лучшей Next-нативностью.
