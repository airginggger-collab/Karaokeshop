## Проект в одном абзаце

Полный ребилд интернет-магазина караоке-оборудования (Алматы; бренды AST и Studio Evolution). Цели: онлайн-продажи (B2C) + B2B-лиды (оснащение заведений) + бренд + **SEO как приоритет**. Архитектура — единая витрина с сегментацией по намерению + B2B-трек «под ключ» (модель B). Стек — Next.js (SSR/SSG) + Headless CMS. i18n-ready (RU сейчас, KZ позже).

## Правила работы с документацией

1. **Один дом.** Всё — в `docs/`. Новый артефакт → в подходящую папку, ссылка добавляется в эту карту.
2. **Прототипы экранов** живут в `design/` (HTML/Figma). Семь стартовых прототипов собраны в чате, на Фазе 1 переносятся в Figma + `design/`.
3. **Решения** фиксируются как ADR в `adr/` (по одному файлу на решение).
4. **Контекст для разработчиков** (модель данных, контракты API, схема контента) накапливается в `context/` по мере Фазы 1 — это то, что отдаётся бэк/фронт-командам.
5. **Коммиты** — только в этот репозиторий, через `git -C ~/Desktop/karaokeshop` (чтобы не перепутать директорию).
