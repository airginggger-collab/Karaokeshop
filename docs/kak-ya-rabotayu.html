1
Интервью с заказчиком
30–60 минут · до открытия редактора
▶

Бизнес и аудитория

Что продаёт, кому, где. B2C или B2B. Текущий сайт и его проблемы.

«Расскажи про бизнес в 3 предложениях. Кто покупает — частники или компании? Что сейчас не работает?»

Одна главная цель

Звонки, заявки, SEO, доверие — выбрать одну. Остальное — потом.

«Если сайт сделает одно — что это? Как ты поймёшь что он работает?»

Что точно НЕ нужно

Исключить заранее: авторизация, рассрочка, корзина, CMS — проговорить до кода.

⚠️ Реальная ошибкаKaspi-рассрочку добавили без спроса — потом удаляли из 21 места по всему сайту. Спроси про запреты ДО начала.
2
Брейншторм и стратегия
1–2 часа · конкуренты, URL-карта, архитектура
▶

Анализ конкурентов

Найти незакрытые позиции в нише и гео. Что делают плохо — сделать лучше.

«Проанализируй конкурентов [ниша] в [город]. Что они делают плохо? Где наше окно?»

URL-карта от семантики

Структура сайта от поисковых запросов, не от «хотелок». Каждый URL = кластер запросов.

«Составь URL-карту от семантического ядра для [ниша + гео]. SEO-приоритет.»

Архитектурное решение

Статика vs SSR, монорепо vs монолит, CMS vs хардкод — с обоснованием каждого выбора.

3
Утверждение визуала
до написания кода · письменное «ок»
▶

Дизайн-токены и палитра

Утвердить цвета и шрифты ДО верстки. Один источник правды — потом не переделывать.

«Предложи 3 варианта палитры для [ниша]. Покажи на кнопке, карточке, хедере.»

Мокап в чате

SVG/HTML мокап прямо в разговоре — утвердить структуру до кода, без Figma.

«Нарисуй мокап главной: hero, 3 секции, футер. Стиль: [референс].»

Письменное «ок»

Заказчик подтверждает конкретный мокап. Скриншот сохраняется в docs/design/.

⚠️ Реальная ошибкаВ karaokeshop дизайн менялся 3 раза (clay morphism → editorial → bento) потому что не было утверждённого мокапа на старте.

Сборка итерациями
скелет → контент → дизайн
▶
ИтерацияФокусРезультат
1 — СкелетМонорепо, роуты, CI/CDЖивой URL на день 1
2 — КонтентВсе страницы, SEO-метаданные, JSON-LDGoogle видит сайт
3 — ДизайнПолировка UI, тёмная тема, анимацииВыглядит как задумано

Правило коммита

Каждый коммит: build ✓ → test ✓ → HANDOFF обновлён → пуш. Никогда не пушить без сборки.

npm run build && npm test → только потом git commit && git push
5
Проверка перед сдачей
чеклист · SEO, мобайл, контент, аналитика
▶
  • ◻
    SEO: Lighthouse ≥ 0.95. Все страницы в sitemap. JSON-LD валидный. Canonical.
  • ◻
    Мобильный: скриншоты 375px и 1280px, обе темы (светлая + тёмная).
  • ◻
    Контент-аудит: grep TODO / FIXME / XXXXXXXX / placeholder по всему коду.
  • ◻
    Аналитика: Umami или Яндекс.Метрика с реальным ID, события фиксируются.
  • ◻
    CTA: все кнопки ведут куда надо — WhatsApp, форма, телефон.
